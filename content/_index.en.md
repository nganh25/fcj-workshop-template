## Internship Project

### Smart Attendance SaaS

**Smart Attendance SaaS** is a multi-organization attendance management system built on Amazon Web Services using a **Serverless** and **Event-Driven** architecture.

During the learning phase, I studied essential AWS services including IAM, Amazon EC2, Amazon VPC, Amazon S3, Amazon CloudFront, Amazon Route 53, Amazon Cognito, Amazon API Gateway, AWS Lambda, Amazon DynamoDB, Amazon SQS, AWS Step Functions, Amazon EventBridge, Amazon SES, Amazon CloudWatch, and Infrastructure as Code tools.

During the project phase, the team analyzed, designed, and implemented a system that allows users to sign in, perform clock-in and clock-out, manage profiles, view attendance history, export reports, receive notifications, and monitor system operations.

My main responsibilities focused on **Cloud Infrastructure, Cloud Networking, AWS service deployment, security, system monitoring, and DevOps support**.

---

## Solution Architecture

The Smart Attendance SaaS architecture includes the following components:

- **Amazon Route 53:** DNS and domain management.
- **Amazon CloudFront:** frontend content distribution.
- **Amazon S3:** storage for the React SPA and report files.
- **Amazon Cognito:** user authentication and JWT issuance.
- **Amazon API Gateway:** managed API access for the frontend.
- **AWS Lambda:** serverless business logic processing.
- **Amazon DynamoDB:** user and attendance data storage.
- **AWS Step Functions and Amazon SQS:** asynchronous task processing.
- **Amazon EventBridge and Amazon SES:** event processing and email notifications.
- **Amazon CloudWatch:** logs, metrics, and operational alarms.
- **AWS CloudFormation and AWS SAM:** Infrastructure as Code deployment.
